Integer和int

发表于 2019-05-03

Integer属于对象，int属于基本数据类型
其次，Integer会存在自动拆箱装箱的过程，也就是int类型和integer类型在程序员看来是可以互换的，但是其中有个缓存池的设定，导致这两者在==判断过程中存在问题

public class Test03 {


   public static void main(String[] args) {

      Integer f1 = 100, f2 = 100, f3 = 150, f4 = 150;


      System. out.println( f1 == f2); //true
      System. out.println( f3 == f4); //false
  }
}

由上面可以看到，当数字在-128~127之间是，integer会指向缓存池中的引用，这也就导致了两个对象指向的地址一致，但是一旦数字超过这个区间，那么就会利用new Integer(150)来在堆上创建对象。

==只是单纯对地址判断的字符，不会判断引用指向的数据，除了常量池中的对象的应用，其他在堆上创建的对象，地址一定不同。

当我们给一个Integer赋予一个int类型的时候会调用Integer的静态方法valueOf。

Integer f1 = Integer.valueOf(100);
Integer f2 = Integer.valueOf(100);
Integer f3 = Integer.valueOf(150);
Integer f4 = Integer.valueOf(150);

思考：那么Integer.valueOf（）返回的Integer是不是是重新new Integer(num);来创建的呢？如果是这样的话，那么== 比较返回都是false，因为他们引用的堆地址不一样。

Interger 赋予的int数值在-128 - 127的时候，直接从cache中获取，这些cache引用对Integer对象地址是不变的，但是不在这个范围内的数字，则new Integer(i) 这个地址是新的地址，不可能一样的.
