静态方法继承的知识点

发表于 2019-05-06

多态分为动态多态和静态多态

  1. 静态多态。重载（依赖参数列表区分）
  2. 动态多态。重写

动态多态

子类继承父类的规则:

  1. 父类的成员方法被子类重写，执行看右边
  2. 子类的静态方法和父类定义一致，执行看左边，编译器就决定的，并不是动态多态
  3. 子类的继承父类的成员变量，但可以覆盖，执行看左边

` 

class Father{
    public String s = "father";
    public static void foo(){
        System.out.println("father 静态方法");
    }
}

class Son extends Father{
    public String s = "son";
    public static void foo(){
        System.out.println("son 静态方法");
    }
}


public class Test {
      public static void main(String[] args){
          Father f = new Son();
          Son s = new Son();
          f.foo();//father 静态方法
          s.foo();//son 静态方法
          System.out.println(f.s);//father
          System.out.println(s.s);//son
          }
}

可以看到，子类继承父类的时候，在代表子类的数据块中，完整保留父类的数据块，包括成员方法，成员变量，静态数据等，但是成员变量又一个动态绑定的特性，而其他没有，因此就依赖静态的声明来区分使用父类的，还是子类的数据块，只有成员方法是看右边的实例判断来动态调用对应的方法。
