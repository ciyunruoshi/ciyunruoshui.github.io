leetcode[4] Median of Two Sorted Arrays

发表于 2019-05-10 | 更新于 2019-05-11

There are two sorted arrays nums1 and nums2 of size m and n respectively.

Find the median of the two sorted arrays. The overall run time complexity should be O(log (m+n)).

You may assume nums1 and nums2 cannot be both empty.

  • Example 1:

nums1 = [1, 3]
nums2 = [2]

The median is 2.0

  • Example 2:

nums1 = [1, 2]
nums2 = [3, 4]

The median is (2 + 3)/2 = 2.5

要求是实现log(m+n)的时间复杂度的程序

分析：

首先log(m+n)的时间复杂度，只有采用二分法来实现了

怎么实现二分法就是重点了，二分法的作用就是不断逼近需要的值，那么我们在这里需要什么

  • 将两个数组组成相等的两个集合，并且左边集合的最大值比右边集合最小值小。
  • 如何在上面的目标上使用二分法

因此：

      left_part          |        right_part
A[0], A[1], ..., A[i-1]  |  A[i], A[i+1], ..., A[m-1]
B[0], B[1], ..., B[j-1]  |  B[j], B[j+1], ..., B[n-1]

这就是我们的目的，当然不考虑极端情况，后续处理。

这样的集合里面满足两个条件

  1. A[i-1]<= B[j]
  2. B[j-1]<= A[j]

###我们一A为主要处理对象，B作为因变量

  • 满足A[i-1]<= B[j] B[j-1]<= A[j]

    已经完成目标集合的构建，接下来就是得到要求的中间值

  • 不满足A[i-1]<= B[j]

说明A[i-1]太大了，A[i-1]需要放到右边集合，而这里面就涉及到一个优化点

当出现这种状况的时候，如何处理减小A的左边最小值，我一开始的想法是，直接i++,j–,然后不断循环，直达得到状况1，但是这样存在缺点：

  • 时间复杂度是O(m+n)而不是log(m+n)

因此不能使用，所以要修改

优化方案

这里就是主要的二分法的使用场景，上面的粗暴想法是发现不满足条件每次修改一个元素，那么我们每次修改一半的元素，这样就可以更快达到需要的集合，并且时间复杂度也满足。

  • 不满足B[j-1]<= A[j]

处理方式和上面一样，就是每次修改一半

代码：

class Solution {
    public double findMedianSortedArrays(int[] nums1, int[] nums2) {
         if(nums1.length>nums2.length){
             int[] temp = nums1;
             nums1 = nums2;
             nums2 = temp;
         }
        int m = nums1.length;//m<n
        int n = nums2.length;
        int imin = 0;
        int imax = m;
        while(imin<=imax){
        //每次重新计算B数组的左右长度
            int i = (imin+imax)/2;
            int j = (m+n+1)/2-i;
            if(i<imax&&nums2[j-1]>nums1[i]){
                imin = i+1;
        //A[i]的值太小了，在右半序列重新寻找合适分界线
            }else if(i>imin&&nums1[i-1]>nums2[j]){                                                                                                                                                                                                                                                                                                                                                                                                                                                                       
                imax = i-1;
        //A[i]的值太大了，在左半序列重新寻找合适分界线
            }else{
                int maxleft=0;
                if(i==0) maxleft = nums2[j-1];
                else if(j==0) maxleft = nums1[i-1];
                else maxleft = Math.max(nums1[i-1],nums2[j-1]);
                if((m+n)%2==1){
                    return maxleft;
                } 

                int minRight = 0;
                if(i==m) minRight=nums2[j];
                else if(j==n) minRight = nums1[i];
                else minRight = Math.min(nums1[i],nums2[j]);

                return (minRight+maxleft)/2.0;
            }
        }
        return 0.0;
    }
}
