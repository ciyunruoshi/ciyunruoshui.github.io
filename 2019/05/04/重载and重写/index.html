重载and重写

发表于 2019-05-04

##重写的规则

  1. 参数列表必须完全和重写方法相同
  2. 返回类型可以不相同，但必须是父类返回类的派生类。
  3. 访问权限不能比父类更低，public>protected>缺省>private
  4. final的方法不能重写
  5. 声明为static的方法不能被重写，但能被重新声明
  6. 构造方法不能被重写

##重载规则

  1. 重载方法必须改变参数列表
  2. 重载方法可以改变返回类型
  3. 重载方法可以改变修饰符
  4. 方法能够在同一个类中或者在一个子类被重载
  5. 返回值不能作为重载的区分标志。

##为什么返回值不作为重载的区分标志呢？
如果可以，从程序编写者的角度来看，我调用方法，Object.foo(7);
这时候我想要使用不同的重载函数，却没用办法利用返回这区分，
这就尴尬了，同样也让程序没法判断我们意图，因此以返回值区分重载函数。

##重写方法为什么要参数不变，其他修饰可以向下兼容？

重载是为了区分同名不同方法，重写则是相反，同名同一方法，而参数列表作为区分不同方法的标志，当然不能变化，但是其他的话不作为判断标志，

其中为什么不能返回类型随意改变？

当返回类型随意变化，同样就有了同名同参数列表方法，但是int 和 long返回类型不同那就是不同的方法，导致子类继承的方法和自己重写的方法不一样，程序就没法判断使用哪个方法
