Aop编程

发表于 2019-05-04

###Aop概念

AOP像OOP一样，只是一种编程范式，AOP并没有规定说，实现AOP协议的代码，要用什么方式去实现。
概念：动态地将代码切入到类的指定方法、指定位置上的编程思想就是面向切面的编程

将公共代码封装到一个可重用模块，这样就能减少系统的重复代码。

###IOC

IOC,也就是控制反转，是一种编程思想，对象的生命周期和作用区域由第三方控制，而不是由程序自己创建，控制权发生转移
DI，依赖注入是最典型的实现方法，由第三方（spring）来负责控制对象的生命周期和对象间的关系。
四种注入方式：

  1. 接口注入

接口注入很好理解：利用反射方式，

public class A{
    InterfaceB B;

    public void doSomething(){
        Object obj = Class.forName(Config.BImplementation).newInstance();
        B = (InterfaceB)B;
        B.doIt();
    }

}

可以看到，利用发射机制来对接口注入它的实现类。

  1. setter方法注入

符合POJO协议的类可以利用setter方法注入到需要的类中

  1. 构造方法注入

利用构造方法注入指定实例对象，而不需要类自己创建

  1. 注解方法注入

Spring中利用@Autowired注解，将实例的创建交给框架，利用注解命令注入需要改对象的类中。
