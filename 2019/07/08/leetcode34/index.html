leetcode34 Find First and Last Position of Element in Sorted Array

发表于 2019-07-08

这道题的重点在于找到数组中相同数字的上下限，而题目要求是logn,那么就只能用二分法，也就是用两个二分法来找上下限制：
code

class Solution {
    public int[] searchRange(int[] nums, int target) {
        int n = nums.length;
        int lo = 0;
        int hi = n;
        while(lo<hi){
            int mid=(lo+hi)/2;
            if(nums[mid]>=target) hi = mid;
            else lo = mid+1;
        }
        int[] out = new int[2];
        out[0]=-1;
        out[1]=-1;

        if(lo==n||nums[lo]!=target)
            return out;

        int s = lo;
        lo = 0;
        hi=n;
        while(lo<hi){
            int mid=(lo+hi)/2;
            if(nums[mid]>target) hi=mid;
            else lo=mid+1;
        }
        out[0]=s;
        out[1]=lo-1;
         return out;
    }
}

这里有一个会被忽视的重点，就是为什么hi 要hi要使用n而不是n-1,

  1. 在找下限的时候，是以hi作为保留target的最高下标，这就导致了，最后得到的下边会在最小的目标元素上
  2. 在找上限的时候，上面的思路就会出错
  • 如果仿照下限去找上限，那就会导致，lo=mid,而这就会出现问题，因为mid一定回事中间值或者是lo值，这是除法决定的，而一旦让mid=lo的话，就会陷入死循环，因为下限的时候是用hi去逼近，不会出现mid=hi的情况，除非lo=hi。
  • 上限不能使用lo去逼近mid,那么只能用hi去逼近，但是hi只能逼近下限，则么办，换个思路，target的上限逼近不了，那就去逼近target下一个值的最小值，也就是代码里的思路

总之，数组逼近的时候，用于两个用途

  • 查找指定的值，则使用mid-1和mid+1
  • 查找范围，一定只能使用hi去逼近，牢记mid=lo 会导致死循环。
