ASCII Unicode UTF-8编码

发表于 2019-03-31

  1. ASCII码

    ASCII是最早的编码，使用一个字节也就是8bit来表示128位英文字符，2^7=128 2^8=256,所以ASII码的最高为全设为0，这样利用ASCII就可以表示所有的英文，非常好用。

  2. Unicode

但是，当网络成为互联网的时候，出现了问题，英文用128个字符就可以，但是法语，德语等就不止128个，于是这些国家把一个字节的8位全部用于编码，中文不够怎么办，按照逻辑可以在增加字节，比如用两个字节表示一个中文符号。

上面问题可以解决的话，传输问题就没法解决了，每个国家都有编码表，中国人用中文编码发送个法国人，法国人按照法语编码就是乱码，没法通行。

于是，国际上就提出Unicode编码用来包含世界上所有的符号，这是国际确认的标准表，对所有字符用二进制表示。

同样这也带来了问题，就是有的英文字符很简单，如a,A只需要一个字节表示，如果使用完整的Unicode码，大量二进制为0，浪费，于是对Unicode码的不同编码规则出现，进过编码后更易传输，减少成本，UTF-8就是编码规则之一。

  1. UTF-8

    上面说了，UTF-8是Unicode的编码传输实现方式之一，采用变长编码。是互联网主要使用的编码。

区别

Unicode 是「字符集」
UTF-8 是「编码规则」

字符集：为每一个「字符」分配一个唯一的 ID（学名为码位 / 码点 / Code Point）
编码规则：将「码位」转换为字节序列的规则（编码/解码 可以理解为 加密/解密 的过程）

PS.要实现Unicode编码，首先要面对的问题就是，如何区分ASCII和Unicode，因为ASCII编码只需要使用一个字节，而Unicode明显要使用多个字节，不同的Unicode编码实现方式不同。
